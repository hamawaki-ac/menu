陸上部 練習メニュー管理　複数人Web共有版

■ 共有仕様
・Firebase Authenticationでコーチごとにログイン
・Cloud Firestoreでメニューを1件ずつ保存
・練習予定を1日ずつ保存
・他のコーチが保存した変更をリアルタイム反映
・週間入力は7日分を一括保存
・最終更新者のメールアドレスを表示
・Firebase未設定時は端末保存で動作

■ Firestore構造
teams
└ athletics-club
  ├ menus
  │ └ {menuId}
  └ days
    └ {YYYY-MM-DD}

■ Firebase設定
1. Firebaseでプロジェクトを作る
2. Webアプリを追加する
3. firebase-config.jsへ設定を貼る
4. Authenticationで「メール/パスワード」を有効化
5. Firestore Databaseを作成
6. firestore.rulesをFirestoreルールへ貼り付けて公開
7. index.htmlとfirebase-config.jsをGitHub Pagesなどへアップロード

■ 注意
現在のルールでは、ログインできたユーザーは全員編集できます。
管理者・編集者・閲覧専用に分ける場合は、メンバーコレクションと権限ルールを追加してください。

■ 基本の時間配分例
月・水・金 16:15〜18:15
16:15〜16:35 全体ウォームアップ
16:35〜16:50 ドリルまたは練習前補強
16:50〜17:45 種目別メイン
17:45〜18:00 補強
18:00〜18:15 クールダウン・連絡

土 9:00〜12:00
9:00〜9:30 全体ウォームアップ
9:30〜9:50 ドリル・動きづくり
9:50〜11:10 種目別メイン
11:10〜11:40 筋トレまたは補強
11:40〜12:00 クールダウン・振り返り

■ 運用上の注意
・90名を能力、学年、種目で小グループに分ける
・中学生は成長差が大きいため、同じメニューでも本数、距離、負荷を個別調整する
・高強度日を連続させず、月水金土の中で負荷を波形化する
・痛みや体調不良がある選手は別メニューへ切り替える


■ 正式運用版
